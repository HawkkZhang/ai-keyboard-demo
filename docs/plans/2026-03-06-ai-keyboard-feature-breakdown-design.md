# AI键盘功能架构设计

> **快速理解：** 本文档为架构设计参考。具体要做什么（93 个子任务 checklist）见根目录 `项目需求文档.md`；分工与里程碑见 `docs/项目启动会.md`。

---

## 项目背景

**产品愿景：** 大模型时代，输入不再只是「字符解码」，而是意图表达。AI 键盘的品牌主张：**每个人的 AI 键盘**。

- 「每个人」——个性化：不仅是装扮，更是键盘功能的自定义能力
- 「AI 键盘」——输入全面 AI 增强 + 主动式 AI 交互

**目标人群：** 18-30 岁年轻用户，社交与内容发布为主，AI 接受度高、爱表达。

---

## 功能架构总览

```
┌─────────────────────────────────────────────────────┐
│              个性化与定制化（横切层）                    │
│   AI服务组件化 · 工具栏自定义 · 场景智能推荐            │
│   AI助手与装扮系统                                     │
├───────────────┬───────────────┬─────────────────────┤
│  前端交互能力   │   模型能力     │     输入功能         │
│ · AI键交互     │ · 端云协同模型  │ · 智能联想与补全      │
│ · 智能体使用   │ · 主Agent路由  │ · 语音输入增强       │
│ · 问AI界面     │ · 智能体调度   │ · 个人信息快速调用    │
│ · 结果展示编辑  │ · 场景意图识别  │ · 智能提醒          │
│ · 场景推荐UI   │ · 信息提取     │                     │
└───────────────┴───────────────┴─────────────────────┘
```

- **个性化横切层**：产品核心差异化，渗透到三个纵向维度
- **三纵**：前端、模型、输入，各自独立，可由不同团队负责

**合计 21 个功能模块**：与 `docs/specs/pm-a~pm-d` 一致

---

## 模块涉及维度（跨团队协作参考）

| 模块 | 涉及维度 | 说明 |
|------|----------|------|
| AI服务组件化、场景智能推荐、AI助手装扮 | 前端 + 模型 | 需前后端协同 |
| 工具栏自定义、AI键、问AI、结果展示、场景推荐UI | 前端 | 以前端为主 |
| 端云协同、主Agent、智能体调度、场景识别、信息提取 | 模型 | 以模型为主 |

---

## 模块依赖关系

```
个性化:
  AI服务组件化框架 ──→ 工具栏自定义、场景智能推荐

前端:
  AI键 / 智能体使用 / 问AI ──→ AI结果展示与编辑

模型:
  端云协同大模型 ──→ 主Agent ──→ 智能体调度
  场景与意图识别 ──→ 场景智能推荐
  个性化信息提取 ──→ 个人信息快速调用、智能提醒

跨维度:
  端云协同大模型 ──→ 智能联想与补全
```
