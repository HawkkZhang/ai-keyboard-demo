# AI键盘项目需求文档

---

## 快速理解（供模型/工具快速加载）

**项目是什么：** 面向 18-30 岁年轻用户的 AI 增强输入法，产品主张「每个人的 AI 键盘」。

**要交付什么：** 个性化定制 + AI 增强输入 + 主动式 AI 交互。核心差异化是键盘功能可自定义（不只是换肤），用户决定「键盘能做什么」。

**具体要做什么：** 22 个功能模块，**按四个工作组组织**，与 `docs/specs/pm-a~pm-d` 及 `docs/项目启动会.md` 一致。执行时按组查找即可。

**文档导航：**
- 本文档：需求清单、子任务 checklist、依赖关系（按工作组）
- `docs/项目启动会.md`：产品定位、品牌、工作组、里程碑
- `docs/specs/pm-a~pm-d`：按工作组的详细功能描述
- `docs/plans/`：功能拆解设计、AI 键 Demo 设计

---

## 目标

**目标：** 交付「每个人的 AI 键盘」——个性化定制 + AI 增强输入 + 主动式 AI 交互。

## 一、按工作组的需求清单（22 模块）

> 与 `docs/specs/pm-a~pm-d` 及 `docs/项目启动会.md` 第 3.1 节一一对应。仅列 specs 中定义的能力。

### 1.1 前端交互组（pm-a，9 个模块）

#### 1.1.1 AI服务组件化框架
需求描述：将所有 AI 服务统一为可插拔组件，作为自定义能力的底座
具体需求：
[] 设计统一的AI服务组件接口规范
[] 将现有功能（助手、智能体、剪贴板、常用语等）改造为组件形式
[] 实现组件注册、发现、加载机制
[] 组件的版本管理与热更新方案

#### 1.1.2 工具栏自定义
需求描述：用户可以自定义键盘工具栏区域显示的AI服务入口
具体需求：
[] 设计工具栏配置界面
[] 实现拖拽排序功能
[] 支持添加/移除组件到工具栏
[] 工具栏布局自适应（根据组件数量调整）
[] 用户配置的云端同步
[] 预设多套工具栏模板供用户快速选择

#### 1.1.3 场景推荐UI
需求描述：根据当前输入场景，智能推荐相应的AI服务入口
具体需求：
[] 推荐结果展示UI设计与实现
[] 智能体推荐卡片展示（单个/多个智能体）
[] 用户反馈机制（点击/忽略推荐的数据收集）
[] 推荐结果的动画效果
[] 推荐位置自适应

#### 1.1.4 AI键交互
需求描述：在按键区域增加AI键，承载高频的AI增强输入功能
具体需求：
[] AI键UI设计与实现（位置、样式、动效）
[] AI键点击交互逻辑（触发联想/补全）
[] AI键状态管理（可用/不可用/加载中）
[] AI键与其他按键的布局适配

#### 1.1.5 智能体使用交互
需求描述：从工具栏启动某个智能体后，键盘区域的交互流程
具体需求：
[] 智能体激活后的键盘状态变化设计
[] 输入区域适配（专属输入引导/预设Prompt展示）
[] 智能体内多轮交互的承接方式
[] 退出智能体/切回普通输入的路径
[] 不同智能体的差异化交互适配

#### 1.1.6 问AI交互界面
需求描述：提供开放性指令入口，用户可以随意表达意图
具体需求：
[] 问AI入口UI设计（工具栏组件形式）
[] 对话界面设计与实现
[] 语音/文本双模输入支持
[] 对话历史记录
[] 问AI的启用/禁用配置
[] AI 结果展示与编辑（生成结果展示、编辑、上屏、流式输出、失败提示）

#### 1.1.7 推荐内容订阅
需求描述：允许用户订阅/取消订阅输入法基于关键词匹配的推荐内容（表情、语录、颜文字），未订阅用户不展示此类推荐
具体需求：
[] 推荐内容订阅设置界面（表情、语录、颜文字可分别订阅）
[] 订阅配置的持久化存储与云端同步
[] 订阅状态与候选区推荐展示的联动（订阅则展示，未订阅则不展示）
[] 新用户默认订阅策略（可选：默认全开/默认全关/首次引导选择）

#### 1.1.8 键盘整体 UI 升级
需求描述：键盘整体视觉与交互的升级，提升现代感与一致性。负责：pm-a + UI/UX
具体需求：
[] 键盘整体布局与视觉规范升级（按键区、候选区、工具栏的协调统一）
[] 动效与过渡效果（键盘唤起/收起、模式切换、状态变化）
[] 与皮肤/主题/装扮的视觉协调
[] 信息层级与可读性优化

#### 1.1.9 语音面板升级
需求描述：语音输入面板的波形展示与交互升级，提升语音输入的视觉反馈与操作体验。负责：pm-a + UI/UX
具体需求：
[] 语音面板波形展示升级（实时波形、音量反馈、识别中状态）
[] 语音录制与松手发送的交互优化
[] 语音面板与键盘整体的视觉统一
[] 语音输入状态（录音中/识别中/完成）的清晰反馈

### 1.2 模型能力组（pm-b，5 个模块）

#### 1.2.1 主Agent意图判断与路由
需求描述：智能分析用户意图，路由到相应的处理模块
具体需求：
[] 主Agent架构设计
[] 意图分类模型/规则（内部指令、外部指令）
[] 路由决策逻辑
[] 多轮对话支持（需要澄清意图时）

#### 1.2.2 智能体调度系统
需求描述：根据任务类型，智能调度专用智能体或通用大模型
具体需求：
[] 智能体匹配算法
[] 智能体调用监控与日志
[] 智能体推荐的个性化

#### 1.2.3 个性化信息提取
需求描述：从用户日常输入中提取关键信息，供后续快速调用和智能提醒
具体需求：
[] 实体识别（人名/昵称/公司等重要实体）
[] 实体关系抽取（亲属关系、生日等）
[] 个人数据识别和提取（卡号/地址/邮箱等）
[] 任务与待办事件识别（近期任务、重要事件）
[] 个人信息库设计与存储

#### 1.2.4 场景推荐策略
需求描述：根据当前场景自动推荐合适的AI服务入口
具体需求：
[] 定义场景到AI服务的映射规则
[] 实现推荐结果与用户自定义配置的融合策略
[] 推荐频次的衰减机制（减少打扰）
[] 推荐结果的个性化学习（基于用户点击/忽略行为）

#### 1.2.5 装扮性格与模型联动
需求描述：将 AI 助手的性格设定转化为大模型 Prompt，实现形象与性格对输出风格的联动
具体需求：
[] 性格模板定义与管理
[] 性格→Prompt 映射规则
[] 形象与性格联动到大模型 Prompt

---

### 1.3 输入能力组（pm-c，6 个模块）

#### 1.3.1 端云协同大模型
需求描述：集成端侧和云端大模型，提供AI增强的输入结果
具体需求：
[] 端侧大模型选型与部署
[] 云端大模型API对接
[] 端云协同策略
[] 模型调用性能优化

#### 1.3.2 场景与意图识别
需求描述：自动识别用户当前的输入场景，驱动场景推荐和 AI 键行为
具体需求：
[] 场景识别能力开发（评论、电商搜索、聊天、内容创作等）
[] 场景与 AI 服务的映射规则配置
[] 文本意图识别能力
[] 意图明确性判断逻辑

#### 1.3.3 智能联想与补全
需求描述：基于端云协同的大模型，提供智能联想和补全功能；支持基于关键词匹配的表情、语录、颜文字推荐（需遵循用户订阅配置）
具体需求：
[] 云端大模型联想接口对接
[] 端侧大模型二遍解码能力
[] 联想结果展示UI（候选词区域）
[] 联想结果的选择与上屏
[] 联想触发时机优化
[] 联想结果缓存机制
[] 基于关键词匹配的表情/语录/颜文字推荐（仅对已订阅用户展示）

#### 1.3.4 语音输入增强
需求描述：语音输入结果支持个性化后处理，可娱乐化也可效率导向
具体需求：
[] 多种后处理风格定义
[] 用户自定义后处理风格
[] 风格切换快捷入口

#### 1.3.5 个人信息快速调用
需求描述：在拼音、语音、指令输入时快速调用已提取的个人信息
具体需求：
[] 拼音输入时的个人信息候选（提取到的个人数据如卡号/地址/邮箱等）
[] 指令输入时的信息自动填充
[] 个人信息编辑与管理界面

#### 1.3.6 智能提醒
需求描述：基于从输入中提取的任务和事件，在临近时间提醒用户
具体需求：
[] 事件时间解析
[] 提醒触发机制
[] 提醒展示方式（通知/键盘内提示）
[] 提醒管理（查看/编辑/删除/过期）

---

### 1.4 装扮能力组（pm-d，2 个模块）

#### 1.4.1 AI助手与装扮系统
需求描述：换装——提供多套预设IP形象供用户选择，IP形象联动键盘皮肤/主题；性格与形象绑定，选择形象即获得对应性格，不支持形象与性格定制；问AI结果页风格与选定形象适配
具体需求：
[] 建设预设IP形象库（形象选择，不支持定制）
[] 实现IP形象与键盘皮肤/主题风格的联动
[] AI助手在交互界面中的视觉呈现与动画效果
[] 问AI结果页的页面风格与选定形象适配
[] 形象切换流程（换装），性格随形象自动绑定

#### 1.4.2 键盘皮肤系统
需求描述：提供多种键盘皮肤与主题，与 IP 形象和装扮风格联动
具体需求：
[] 预设皮肤库设计
[] 皮肤与主题切换
[] 皮肤与 IP 形象联动

## 二、模块依赖关系

```
前端组:
  AI服务组件化框架 ──→ 工具栏自定义
  AI服务组件化框架 ──→ 场景推荐UI
  推荐内容订阅 ──→ 智能联想与补全（输入组，订阅配置决定是否展示表情/语录/颜文字）

模型组:
  端云协同大模型 ──→ 主Agent意图判断与路由
  主Agent ──→ 智能体调度系统
  场景与意图识别（输入组）──→ 场景推荐策略 ──→ 场景推荐UI（前端组）
  个性化信息提取 ──→ 个人信息快速调用（输入组）
  装扮性格与模型联动 ←── AI助手与装扮系统（形象选择时性格已绑定，模型组负责性格→Prompt）

输入组:
  端云协同大模型 ──→ 智能联想与补全
  端云协同大模型 ──→ 场景与意图识别
  个性化信息提取（模型组）──→ 个人信息快速调用
  个性化信息提取 ──→ 智能提醒
```
